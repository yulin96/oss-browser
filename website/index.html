使用流程

连接账号并定位对象

客户端直接调用阿里云 OSS 接口，不经过项目自建的中转服务。账号配置与界面状态保存在本机。

下载最新版本 →
  1. 01
    配置访问凭证

    添加 AccessKey、STS 凭证或临时授权码，并配置对应的 Endpoint。

  2. 02
    选择 Bucket 与路径

    从 Bucket 列表进入，或输入 oss://bucket/path/ 地址定位对象前缀。

  3. 03
    执行 OSS 操作

    上传、下载或管理对象，并按需使用预览、临时 URL、ACL 和 CDN 刷新功能。

实现方式

项目说明

这是一个可本地运行、可自行构建的 Electron 项目。仓库包含应用源码、打包配置和跨平台发布工作流。

01 / STACK

Electron + Vue 3 + TypeScript

Electron 提供桌面能力，Vue 3 负责渲染界面，TypeScript 用于主进程、预加载脚本与渲染进程代码。

02 / PROCESS

主进程隔离系统能力

OSS 请求、凭证访问和文件系统操作保留在主进程，通过受控的预加载接口提供给界面。

03 / LOCAL DATA

账号数据保存在本地

首页位置、收藏、显示设置和传输记录按账号存储；保存的凭证由 Electron safeStorage 保护。

04 / RELEASE

仓库内完成多平台构建

使用 pnpm 执行开发、检查与打包任务，GitHub Actions 为 macOS、Windows 和 Linux 生成发布包。

查看开发与构建说明 →
本地安全边界

凭证与本地数据处理

保存的凭证通过 Electron safeStorage 调用操作系统密钥存储能力加密。账号首页、收藏、显示设置与传输记录按账号写入本地应用数据目录。

OSS Browser 是独立维护的第三方项目，并非阿里云官方客户端。发布包由仓库中的 GitHub Actions 构建并上传到 GitHub Releases。
